## 全局结构问题分析

### 4. 文档结构混乱
- **表现**：
  - 顶层 `documents/` 与 `documents/archive/` 的职责界限模糊，历史记录与现行指南混杂。
  - 同一主题出现多份版本（如图像预处理相关指南重复），信息冗余。
  - 缺乏统一的目录索引或导航，难以引导新成员快速定位资料。
- **结论**：⚠️ 问题存在，属于结构性改进重点。
- **建议下一步**：
  1. 建立总览索引文档，说明各类文档的用途与位置。
  2. 明确归档标准（例如：已完成/历史内容转入 `archive/`，现行指南保留在根目录）。
  3. 合并或互相引用重复主题文档，避免信息分散。
  4. 在调整前备份并规划迁移顺序，确保 Git 历史可追溯。

### 5. 文档内容冲突与缺失
- **表现**：
  - 部分文档对参数或流程的描述，与最新代码实现存在偏差（如 `config.py` 新增项未同步到指南）。
  - 多份 `image_preprocessing` 系列文档互相交叉，但未明确最终推荐流程。
  - `aimassistant_intro_for_ai.md` 与相关 Journey 文档对自瞄流程的细节表述存在不一致。
- **结论**：⚠️ 问题存在并需整治。
- **建议下一步**：
  1. 为关键模块（自瞄、识别、预处理、硬件调试）建立“代码 vs 文档”对照表，列出差异点。
  2. 制定文档更新 Checklist，规定改动代码时必须同步更新相关文档。
  3. 明确“权威信息来源”（例如配置说明以 `config.py` 为准），纠正分歧内容。
  4. 安排至少一次团队审阅，减少个人理解偏差。
